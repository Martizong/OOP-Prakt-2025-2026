## Структури
Структурите в C++ са съставен тип данни. Използват се за групиране на елементи.
Елементите могат да бъдат от различен тип и с различна дължина. Фундаментални типове данни - int, double, bool, char,..
Последното е точно това, което различава структурите от масивите.
В масивите съхраняваме само хомогенни данни (такива от един и същ тип).

```c++
struct Point
{
       int x;
       int y;
}
```

```c++
struct Person
{
	char firstName[32];
	char lastName[32];
	unsigned int age;
	char ucn[11]; // unified civilian number
};
```

# Деклариране на обекти от новия тип

```c++
Point A; //default values to x and y
Point B = {3, 4} // x = 3, y = 4;
Point C;
C.x = -11;
C.y = -100;

Person p; // default values are assigned
Person p2{ "Ivan", "Dimitrov", 25, "9612120305" };
// firstName == Ivan, lastName == Dimitrov, age == 25, ucn == 9612120305
Person p3;
strcpy(p3.firstName, "Ivan");
strcpy(p3.lastName, "Dimitrov");
p3.age = 25;
strcpy(p3.ucn, "9612120305");
```

# Подаване във функции

- Ако няма да променяме обекта го подаваме по константна референция.

```c++
int distance(const Point& q,const Point& p)
{
  	double dx = q.x - p.x;
   double dy = q.y - p.y;

   return sqrt(dx*dx + dy*dy);
}
```

- Може и само по референция, но тогава промените ще се отразят върху подадения аргумент.
  
```c++
void readPoint(Point& D)
{
    cin >> D.x >> D.y;
}
```

- Може и да го подаваме по копие.

```c++
Point revertPoint(Point F)
{
    int temp = F.x;
    F.x = F.y;
    F.y = temp;

    return F;
}
```

## Разположение в паметта

- Елементите на всяка структура "вървят в комплект". Те се записват последователно в паметта точно в реда, в който сме ги изредили в структурата.

- Подравняване (alignment) - Поради начина на работа на процесора, имаме следното правило:
Ако T е типът на променливата V, то V трябва да бъде разположена на адрес, кратен на изискваното подравняване за T, т.е. address(V) % alignof(T) == 0.
Когато работим с фундаментални типове, компилаторът се грижи за коректното им подравняване.
Когато работим със структури, ние трябва да осигурим оптималното "подреждане" на елементите им, имайки предвид подравняванията.

Пример:

```c++
struct Test
{
	int i1;
	char c;
	int i2;
};
// На пръв поглед, за създаване на инстанция на структурата Test са ни нужни 9 байта памет.
// Но само на пръв поглед.
// Т.е. за създаване на такава инстанция са ни нужни 4 (sizeof(i1)) + 1 (sizeof(c)) + 3 (padding) + 4 (sizeof(i2)) = 12 байта.
```

## Битови полета
Битовите полета позволяват членове на структура да заемат точен брой битове, вместо стандартни байтове.
Използват се за компактно съхранение на флагове, маски, хардуерни регистри.
```c++
struct Flags {
    unsigned int isVisible : 1;   // 1 бит
    unsigned int isActive  : 1;   // 1 бит
    unsigned int mode      : 2;   // 2 бита (стойности 0–3)
};
```

Нормално int заема 4 байта (32 бита). При битово поле можеш да кажеш: Тази променлива използва само 1 бит.
Тук структурата съдържа три стойности, но те заемат само 4 бита общо, вместо минимум 12 байта. Това пести памет.

## Union

- позволява няколко членове да споделят една и съща памет.

- Само един член е валиден в даден момент. Oбединението има размера на най-големия тип данни в полетата му

- Обединението е като една кутия, в която можеш да сложиш различни типове данни, но само едно нещо наведнъж.
 
## Разлика спрямо struct

- При struct  всеки член има собствена памет
  
- При union  всички членове ползват една и съща памет

## Енъми
Enum е начин да дефинираме множество от именувани стойности, които представляват цели числа. Използват се за по-ясен и четим код, особено когато работим със състояния, опции или флагове.

- Декларираме enum по следния начин:

```c++
enum <name>
{
    <enumerator1>,
    <enumerator2>,
    <enumerator3>,
    ...
};
```

- Стойностите на енумераторите са нарастващи, започващи от 0 и от първия елемент.

```c++
enum Color
{
    Red,    // 0
    Green,  // 1
    Blue    // 2
};

int main()
{
    Color color = Green;
    cout << color; // 1
}
```
## Основни неща:
1. Не можем да имаме два изброими типа, които съдържат повтарящи се енумератори

 ```c++
enum Gender { Male, Female };
enum Gender2 { Male, Female };   // проблем
 ```

2. Не можем да създадем променлива с име, което вече съществува като енумератор в някой enum

 ```c++
int Male = 5;   // ако има enum с Male, конфликт
 ```

3. При plain enums имаме имплицитно конвертиране към int, което води до проблеми

 ```c++
int main()
{
    Color color = Color::Red;

    Animal animal = Animal::Bird;

    int num = color;          // имплицитно към int

    if (color == Animal::Dog)   // позволено (но логически грешно)
        ...

    if (animal == Color::Purple)   // позволено
        ...

    enum Color { RED, GREEN };
    enum Size  { SMALL, LARGE };

    if (RED == SMALL) { } // компилира се (но е безсмислено)
 }
```

# Enum class - Това е силно типизиран изброен тип.

 ```c++
enum class Color { RED, GREEN, BLUE };
Color c = Color::RED;

int x = Color::RED; // ГРЕШКА
(трябва static_cast<int>(Color::RED))

 ```
# Предимства

- Стойностите са вътре в типа:

 ```c++
Color::RED
Color::GREEN
 ```

- Няма конфликти:

 ```c++
enum class Color { RED };
enum class TrafficLight { RED }; // няма проблем
 ```
- Няма безмислени сранения

```c++
enum class Color { RED };
enum class Size  { SMALL };

if (Color::RED == Size::SMALL) //ГРЕШКА 
```

# Обобщение
Обикновеният enum е слабo типизиран и може да доведе до грешки чрез неявни преобразувания и конфликти на имена.
enum class въвежда силна типизация и локализира имената, което прави кода по-безопасен и предвидим.
