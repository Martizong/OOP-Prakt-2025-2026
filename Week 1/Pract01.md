## Динамична памет

- Динамична памет - може да се задели или освободи памет по всяко време на изпълнение на програмата

- Областта за динамична памет е набор от свободни блокове памет;

- Програмата може да заяви блок с произволна големина;

- Имаме контрол над управлението на паметта;
  
- оператор new - задаля динамична памет

- new <тип> - заделя блок от памет за една променлива

- new <тип>[брой] - заделя блок от памет за масив, с размер подадения брой

- оператор delete - освобождава динамично заделената памет

- delete <указател> - освобождава блок от памет с начало, сочено от указателя

- delete[] <указател> - освобождава блок от памет съдържащ масив, с размер подадения брой

- memory leak - не е освободена памет, заделена с new

## Разлика между указател и референция?

- Указателят може да се "re-assign"-ва. Може да приема нови стойност
  
- Референцията трябва да се инициализира при създаването.

- Указателят има неутрална стойност (nullptr), а референцията няма.

- Може да се направи масив от указатели, но не и масив от референции.

## Указатели към функции
Указател към функция е променлива, която пази адреса на дадена функция и позволява тя да бъде извикана по-късно чрез този адрес.
За да можем да използваме такъв указател, той трябва да има същата сигнатура като функцията (същите параметри и тип на връщане).
Това ни позволява да подаваме функции като параметри на други функции и да избираме коя логика да се изпълни по време на програмата.
За разлика от указателите към данни, те сочат към код, а не към заделена памет.


## Побитови операции
Прилагат се върху един бит или набор от повече отделни битове на двоични числа.

 - Побитово И (**&**)    

|        |           |   
|--------|-----------|   
| a      | 1001 0110 |   
| b      | 1010 1100 |
| a&b    | 1000 0100 |


 - Побитово ИЗКЛЮЧВАЩО ИЛИ  (**^**)

|        |           |
|--------|-----------|
| a      | 1001 0110 |
| b      | 1010 1100 |
| a&b    | 0011 1010 |

 - Побитово отместване (**<<**) (**>>**)


|        |           |
|--------|-----------|
| a      | 1001 0110 |
| a << 2 | 0101 1000 |
| a >> 2 | 0010 0101 |

- Побитовa негация (**~**) 


|    |           |
|----|-----------|
| a  | 1001 0110 |
| ~a | 0110 1001 |



## Изброен тип
Enum е начин да дефинираме множество от именувани стойности, които представляват цели числа. Използват се за по-ясен и четим код, особено когато работим със състояния, опции или флагове.

- Декларираме enum по следния начин:

```c++
enum <name>
{
    <enumerator1>,
    <enumerator2>,
    <enumerator3>,
    ...
};
```

- Стойностите на енумераторите са нарастващи, започващи от 0 и от първия елемент.

```c++
enum Color
{
    Red,    // 0
    Green,  // 1
    Blue    // 2
};

int main()
{
    Color color = Green;
    cout << color; // 1
}
```
## Основни неща:
1. Не можем да имаме два изброими типа, които съдържат повтарящи се енумератори
enum Gender { Male, Female };
enum Gender2 { Male, Female };   // проблем

2. Не можем да създадем променлива с име, което вече съществува като енумератор в някой enum
int Male = 5;   // ако има enum с Male → конфликт

3. При plain enums имаме имплицитно конвертиране към int, което води до проблеми

 ```c++
int main()
{
    Color color = Color::Red;

    Animal animal = Animal::Bird;

    int num = color;          // ✔ имплицитно към int

    if (color == Animal::Dog)   // ✔ позволено (но логически грешно)
        ...

    if (animal == Color::Purple)   // ✔ позволено
        ...
}
```
